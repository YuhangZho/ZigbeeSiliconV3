104 /* @brief LEGACY FUNCTION: This functionality has been replaced by a single
105  * bit in the ::EmberApsFrame, ::EMBER_APS_OPTION_DSA_SIGN. Devices wishing
106  * to send signed messages should use that as it requires fewer function calls
107  * and message buffering. emberDsaSignHandler() is still called when using
108  * ::EMBER_APS_OPTION_DSA_SIGN. However, this function is still supported.
109  *
110  * This function begins the process of signing the passed message
111  * contained within the buffer. If no other ECC operation is going on,
112  * it will immediately return with ::EMBER_OPERATION_IN_PROGRESS.
113  * It will delay a period of time to let APS retries take place, but then it
114  * will shutdown the radio and consume the CPU processing until the signing
115  * is complete. This may take up to 1 second.
116  *
117  * The signed message will be returned in ::emberDsaSignHandler().
118  *
119  * Note that the last byte of the buffer contents passed to this function has
120  * special significance. As the typical use case for DSA signing is to sign the
121  * ZCL payload of a DRLC Report Event Status message in SE 1.0, there is often
122  * both a signed portion (ZCL payload) and an unsigned portion (ZCL header).
123  * The last byte in the content of messageToSign is therefore used as a
124  * special indicator to signify how many bytes of leading data in the buffer
125  * should be excluded from consideration during the signing process. If the
126  * signature needs to cover the entire buffer (all bytes except last one),
127  * the caller should ensure that the last byte of the buffer contents is 0.
128  * When the signature operation is complete, this final byte will be replaced
129  * by the signature type indicator (0x01 for ECDSA signatures), and the
130  * actual signature will be appended to the buffer after this byte.
131  *
132  * @param messageToSign The message buffer containing the complete message,
133  * both the to-be-signed portion as well as any leading data excluded from
134  * the signing operation. See note above regarding special requirements
135  * for this buffer.
136  *
137  * @return ::EMBER_OPERATION_IN_PROGRESS if the stack has queued up the
138  * operation for execution. ::EMBER_INVALID_CALL if the operation can't be
139  * performed in this context (possibly because another ECC operation is
140  * pending.)
141  */
142 EmberStatus emberDsaSign(EmberMessageBuffer messageToSign);

197 /* @brief This function is used to update the Smart Energy certificate,
198  * CA public key, and local private key that the device uses for CBKE.
199  * The preferred method for adding certificates is to pre-install them
200  * in MFG tokens when the chip is manufactured. However this function
201  * allows the certificate to be updated at runtime after the device has
202  * been deployed.
203  * The behavior of this function differs based on the hardware platform.
204  *
205  * For the 2xx:
206  * To use this functionality the application must also set
207  * the stack configuration value ::EMBER_CERTIFICATE_TABLE_SIZE to 1.
208  * Attempts to call this function with ::EMBER_CERTIFICATE_TABLE_SIZE of 0
209  * will return ::EMBER_SECURITY_CONFIGURATION_INVALID.
210  * The passed security data will be persistently stored in stack tokens.
211  * The certificate contains the EUI64 it is associated with. If that
212  * EUI64 matches the EUI64 currently in use by the device, this
213  * function may be called at any time, even while running in a network.
214  * If the EUI64 does not match, this function may only be called when the
215  * network is in a state of ::EMBER_NO_NETWORK. Attempts to do otherwise
216  * will result in a return value of ::EMBER_INVALID_CALL.
217  *
218  * For the 3xx:
219  * This function allows a one-time write of the MFG token if it has not
220  * already been set. It does NOT utilize the ::EMBER_CERTIFICATE_TABLE_SIZE
221  * so that should remain set at 0. Attempts to write the certificate that
222  * has already been written will return a result of
223  * ::EMBER_ERR_FLASH_WRITE_INHIBITED. If the EUI64 in the certificate is
224  * the same as the current EUI of the device then this function may be called
225  * while the stack is up. If the EUI in the certificate is different than
226  * the current value, this function may only be called when the network is in
227  * a state of ::EMBER_NO_NETWORK. Attempts to do otherwise will result in a
228  * return value of ::EMBER_INVALID_CALL. If the EUI in the certificate is
229  * different than the current value this function will also write the
230  * Custom EUI64 MFG token. If that token has already been written the operation
231  * will fail and return a result of ::EMBER_BAD_ARGUMENT.
232  * If all the above criteria is met the token will be written and
233  * ::EMBER_SUCCESS will be returned.
234  *
235  * @note The device will immediately and persistently <b>change its EUI64
236  * to match the value in the certificate</b>.
237  *
238  * @param caPublic A pointer to the CA public key data that will be stored
239  * in stack tokens.
240  * @param myCert A pointer to the certificate data that will be stored in
241  * stack tokens.
242  * @param mykey A pointer to the private key data that will be stored in
243  * stack tokens.
244  * @return The ::EmberStatus value indicating success or failure of the
245  * operation.
246  */
247 EmberStatus emberSetPreinstalledCbkeData(EmberPublicKeyData* caPublic,
