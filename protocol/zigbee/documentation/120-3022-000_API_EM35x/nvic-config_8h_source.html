53 // The purpose of this file is to consolidate NVIC configuration, this
54 // includes basic exception handler (ISR) function definitions, into a single
55 // place where it is easily tracked and changeable.
56 //
57 // \b NOTE Our chips only implement 5 bits for priority and subpriority
58 // (bits [7:3]). This means that the lowest 3 bits in any priority field will
59 // be forced to 0 and not used by the hardware
60 //
61 // We establish 8 levels of priority (3 bits), with 4 (2 bits) of tie break
62 // subpriority. Lower priority values are higher priorities. This means that
63 // 0 is the highest and 7 is the lowest. The NVIC mapping is detailed below.
64 //
65 
66 //The 'PRIGROUP' field is 3 bits within the AIRCR register and indicates the
67 //bit position within a given exception's 8-bit priority field to the left of
68 //which is the "binary point" separating the preemptive priority level (in the
69 //most-significant bits) from the subpriority (in the least-significant bits).
70 //The preemptive priority determines whether an interrupt can preempt an
71 //executing interrupt. The subpriority helps choose which interrupt to run if
72 //multiple interrupts with the same preemptive priority are active at the same
73 //time. If no supriority is given or there is a tie then the hardware defined
74 //exception number is used to break the tie.
75 //
76 //The table below shows for each PRIGROUP value (the PRIGROUP value is the
77 //number on the far left) how the priority bits are split and how this maps
78 //to the priorities on chip. A 'P' is preemption bit, 'S' is a subpriority bit
79 //and an 'X' indicates that that bit is not implemented.
80 //0=7:1= PPPPPXX.X
81 //1=6:2= PPPPPX.XX
82 //2=5:3= PPPPP.XXX
83 //3=4:4= PPPP.SXXX
84 //4=3:5= PPP.SSXXX
85 //5=2:6= PP.SSSXXX
86 //6=1:7= P.SSSSXXX
87 //7=0:8= SSSSSSXXX
88 //
89 //Below is the default priority configuration for the chip's interrupts
90 // Pri.Sub Purpose
91 // 0.0 faults (highest)
92 // 1.0 not used
93 // 2.0 SysTick for idling, management interrupt for XTAL biasing,
94 // and the SVCall
95 // 3.0 DISABLE_INTERRUPTS(), INTERRUPTS_OFF(), ATOMIC()
96 // 4.0 normal interrupts
97 // 5.0 not used
98 // 6.0 not used
99 // 7.0 debug backchannel, PendSV
100 // 7.31 reserved (lowest)
101 #if defined(FREE_RTOS)
102 // FreeRTOS recommends this type of configuration for CortexM3 and asserts
103 // in some places if it finds a different one.
104  #define PRIGROUP_POSITION (2) // PPPPP.XXX
105 #else
106  #define PRIGROUP_POSITION (4) // PPP.SSXXX
107 #endif
