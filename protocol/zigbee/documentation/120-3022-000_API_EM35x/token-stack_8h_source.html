70 // The basic TOKEN_DEF macro should not be used directly since the simplified
71 // definitions are safer to use. For completeness of information, the basic
72 // macro has the following format:
73 //
74 // TOKEN_DEF(name,creator,iscnt,isidx,type,arraysize,...)
75 // name - The root name used for the token
76 // creator - a code used to uniquely identify the token when using SimEE
77 // nvm3key - a key used to uniquely identify the token when using NVM3
78 // iscnt - a bool flag that is set to identify a counter token
79 // isidx - a bool flag that is set to identify an indexed token
80 // type - the basic type or typdef of the token
81 // arraysize - the number of elements making up an indexed token
82 // ... - initializers used when reseting the tokens to default values
83 //
84 //
85 // The following convenience macros are used to simplify the definition
86 // process for commonly specified parameters to the basic TOKEN_DEF macro.
87 // DEFINE_BASIC_TOKEN(name, type, ...)
88 // DEFINE_INDEXED_TOKEN(name, type, arraysize, ...)
89 // DEFINE_COUNTER_TOKEN(name, type, ...)
90 // DEFINE_FIXED_BASIC_TOKEN(name, type, address, ...)
91 // DEFINE_FIXED_INDEXED_TOKEN(name, type, arraysize, address, ...)
92 // DEFINE_FIXED_COUNTER_TOKEN(name, type, address, ...)
93 // DEFINE_MFG_TOKEN(name, type, address, ...)
94 //
95

EMBER_CERTIFICATE_TABLE_SIZE
#define EMBER_CERTIFICATE_TABLE_SIZE
The number of entries for the field upgradeable certificate table. Normally, certificates, such as SE certs, are stored in the runtime-unmodifiable MFG area. However, those devices wishing to add new certificates after manufacturing will have to use the normal token space. This defines the size of that table. For most devices 0 is appropriate since there is no need to change certificates in the field. For those wishing to field upgrade devices with new certificates, 1 is the correct size. Anything more is wasting SimEEPROM.
